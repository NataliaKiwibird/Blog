Recursion

Recursion in Javascript is a function that calls itself. It's used to solve problems by breking them into smaller problems. There's a story called Martin and the Dragon that illustrates this process (story available here). Recursive functions occur in many places, such as JSON, DOM traversal algorithms, object traversal and complex data structures. It takes a problem and does its parts over and over again (by recursive call) until it hits a set endpoint (known as a base case).

Base case

A piece of code that sets the point where the function should finish. without a base case, the function runs forever and causes computer freezing.

Recursive call

A piece of code that determines what the function does and how works.

Stack overflow error

It's a runtime error occurring if a base case is not set or set unproperly. It's caused when the call stack's size exceeds the amount of memory reserved for it.

Call stack

A built-in data structure that stores information and manages functions and their calls. It can be compared to a stack of papers - it works in a similar way: When a function is called, it's placed on the top of the stack. When a program process the return keyword or when the function ends the function call is removed from the top of the stack.
