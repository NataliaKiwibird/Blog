CSS Units

CSS uses many dimension units, which can be confusing for beginning developers. Luckily, we can group those units into parts and differ them within their groups. Below I present a simplified table of css units, but to be perfectly clear about css dimensions, we need to first understand the concept of box-sizing property:

box-sizing: content-box;

Default value.
The given dimension includes the width and height of the content only, it doesn't include margins, padding and borders. Margins, paddings, and borders are extra added to the content.

box-sizing: border-box;

Recommended value.
The given dimension includes the content with padding, margins, and borders. This value is easier to maintain on the webpage layout.

Table of CSS Units:

UNIT TYPE DESCRIPTION
EM relative to a font-size Relative to a font-size of a parent element.
It can be used to set other properties than font-size (margin, padding etc.)
REM relative to a font-size Relative to a font-size of a root element.
It can be used to set other properties than font-size (margin, padding etc.)
VH relative to a viewport Relative to the height of the viewport.
VW relative to a viewport Relative to the width of the viewport.
VMIN relative to a viewport Relative to the shorter dimension of the viewport.
Vertical screen = VMIN relative to the viewport's width, Horizontal screen = VMIN relative to the height.
VMAX relative to a viewport Relative to the longer dimension of the viewport.
Vertical screen = VMAX relative to the viewport's height, Horizontal screen = VMAX relative to the width.
% relative to a parent Takes a percentage of the closest parent element size.
PX not relative to anything else Absolute unit. Not recommended in responsive web design.
Recommended for print style sheets.
CM, MM, IN not relative to anything else Absolute unit. Not recommended in responsive web design.
Recommended for print style sheets.
CALC() Connects different units into one length,
e.g. width: calc( 10vw - 10px );
0, line-height, opacity Some of css properties doesn't need a unit.
